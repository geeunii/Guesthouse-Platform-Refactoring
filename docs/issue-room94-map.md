# 방 상세 지도 이슈 (room/94)

## 요약
- 증상: `/room/94`에서 지도 영역이 흰 화면 + 마커만 보임. `/room/1`은 정상 표시됨.
- 범위: `frontend/src/views/home/RoomDetailView.vue`의 방 상세 지도 렌더링.
- 영향: 좌표가 없는 숙소는 지도가 비어 보임.

## 원인
- DB 시드에서 숙소 `94`의 `latitude`/`longitude`가 `NULL`.
- 프론트에서 `Number(null)`을 사용해 좌표 유효성 확인 → `0`으로 변환됨.
- `0`은 유한 값이라 좌표가 있는 것으로 판단됨.
- 결과적으로 `(0, 0)`으로 지도가 생성되어 타일이 로드되지 않고 흰 화면 + 마커만 표시됨.

## 해결
- `null`/빈 값은 좌표로 처리하지 않도록 조건 강화.
- `latitude`와 `longitude`가 모두 유효한 경우에만 좌표 사용.
- 좌표가 없으면 주소 기반 지오코딩으로 지도 중심을 계산.

## 변경 파일
- `frontend/src/views/home/RoomDetailView.vue`

## 비고
- 주소가 너무 부정확한 경우(예: "346-9") 지오코딩이 실패할 수 있음.
- 장기적으로는 DB에 유효한 좌표를 채우는 것이 가장 확실함.
