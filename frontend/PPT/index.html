동시성 제어 — Redis 분산 락

❌ 문제 상황

[기존 비관적 락 테스트]
정원: 6명
동시 요청: 10명

결과: 10명 전원 예약 성공 😱
→ 4명 초과 예약 버그!

MySQL REPEATABLE READ 격리 수준에서
동시 트랜잭션이 서로의 INSERT를 볼 수 없음

✅ 해결: Redis 분산 락

@DistributedLock(key = "'reservation:room:' 
  + #requestDto.roomId() 
  + ':date:' + #requestDto.checkin()")
public ReservationResponseDto createReservation(...)

테스트 결과

10명 동시 요청 → 6명 성공, 4명 실패 ✅

성능 최적화 — 검색 속도 개선

Before

  • 클라이언트 사이드 필터링
  • 전체 데이터 조회 후 JS로 필터
  • 페이지네이션 불일치
  • 대용량 데이터 성능 저하
~500ms
→

After

  • 서버 사이드 필터링
  • DB에서 조건에 맞는 데이터만 조회
  • 정확한 페이지네이션
  • DB 인덱싱 활용
~50ms
응답 속도 10배 ↑
DB 쿼리 최적화
사용자 경험 체감 속도 향상
